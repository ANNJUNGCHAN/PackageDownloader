# PyQt UI 기획서

## 1. 개요

### 1.1 배경
현재 오프라인 패키지 다운로더는 CLI(명령줄) 기반으로 동작합니다. CLI에 익숙하지 않은 사용자도 쉽게 사용할 수 있도록 그래픽 사용자 인터페이스(GUI)를 개발합니다.

### 1.2 목적
- 명령어를 몰라도 클릭만으로 패키지 다운로드 가능
- 시각적 피드백으로 진행 상황을 직관적으로 파악
- 설정을 기억하여 반복 작업 간소화

### 1.3 기대 효과
- 사용자 진입 장벽 낮춤
- 작업 실수 감소 (입력 검증, 시각적 확인)
- 작업 효율 향상 (마우스 클릭 기반 조작)

---

## 2. 사용 흐름

### 2.1 기본 시나리오

**단계 1: 환경 선택**
- 사용자가 프로그램을 실행하면 저장된 환경 목록이 드롭다운으로 표시됨
- 환경이 없으면 "환경을 먼저 등록하세요" 안내 메시지 표시
- 환경 선택 시 해당 환경의 OS, Python 버전, 플랫폼 정보가 하단에 표시됨

**단계 2: 환경 등록 (최초 사용 시)**
- "추가" 버튼 클릭 시 환경 등록 팝업이 열림
- OS 종류를 라디오 버튼으로 선택 (RedHat 계열, Ubuntu 계열, Windows, macOS)
- 선택한 OS 종류에 따라 세부 버전 드롭다운이 변경됨
- Python 버전을 드롭다운에서 선택
- 선택 즉시 하단에 자동 변환된 플랫폼 태그가 미리보기로 표시됨
- "저장" 클릭 시 환경이 등록되고 메인 화면의 드롭다운에 추가됨

**단계 3: 패키지 목록 입력**
- 방법 A: "찾아보기" 버튼으로 requirements.txt 파일 선택
- 방법 B: 텍스트 영역에 패키지 이름을 직접 입력 (줄 단위)
- 파일을 드래그앤드롭으로 끌어다 놓을 수도 있음

**단계 4: 출력 설정**
- "찾아보기" 버튼으로 결과물 저장 폴더 선택
- 압축 생성 여부 체크박스 선택
- 분할 압축 시 분할 크기(MB) 입력

**단계 5: 다운로드 실행**
- "다운로드" 버튼 클릭
- 진행률 바가 채워지며 현재 다운로드 중인 패키지명 표시
- 각 패키지별 성공/실패 상태가 목록으로 실시간 표시
- 다운로드 중 "취소" 버튼으로 중단 가능

**단계 6: 결과 확인**
- 완료 시 결과 팝업 표시 (성공/실패 개수, 총 용량, 소요 시간)
- "폴더 열기" 버튼으로 결과물 폴더를 탐색기에서 열기
- "리포트 보기" 버튼으로 상세 리포트 확인

## 3. 화면 구성

### 3.1 메인 화면

메인 화면은 위에서 아래로 순차적인 작업 흐름을 따르는 단일 화면 구성입니다.

| 영역 | 위치 | 설명 |
|------|------|------|
| 환경 선택 | 상단 | 환경 드롭다운, 추가/편집 버튼, 환경 정보 표시 |
| 패키지 입력 | 중상단 | 파일 선택 또는 직접 입력 텍스트 영역 |
| 출력 설정 | 중하단 | 출력 폴더 선택, 압축 옵션 |
| 진행 상황 | 하단 | 진행률 바, 현재 작업, 로그 목록 |
| 액션 버튼 | 최하단 | 다운로드, 취소 버튼 |
| 상태 표시줄 | 바닥 | 현재 상태 메시지 |

### 3.2 환경 등록 팝업

환경을 새로 등록하거나 수정할 때 열리는 팝업입니다.

| 영역 | 설명 |
|------|------|
| 환경 이름 | 사용자가 입력하는 식별용 이름 |
| OS 종류 | 라디오 버튼으로 대분류 선택 |
| OS 버전 | 선택한 종류에 따른 세부 버전 드롭다운 |
| Python 버전 | 지원 버전 드롭다운 |
| 플랫폼 미리보기 | 선택에 따라 자동 계산된 플랫폼 태그 표시 |
| 저장/취소 버튼 | 하단 |

### 3.3 결과 팝업

다운로드 완료 후 표시되는 팝업입니다.

| 영역 | 설명 |
|------|------|
| 완료 아이콘 | 성공/부분실패에 따라 다른 아이콘 |
| 통계 정보 | 총 패키지 수, 성공/실패 수, 총 용량, 소요 시간 |
| 출력 위치 | 결과물이 저장된 경로 |
| 액션 버튼 | 폴더 열기, 리포트 보기, 확인 |

## 4. 입력 명세

### 4.1 환경 등록 입력

| 입력 항목 | 필수 | 입력 방식 | 검증 조건 |
|----------|-----|----------|----------|
| 환경 이름 | O | 텍스트 입력 | 영문, 숫자, -, _ 만 허용. 중복 불가 |
| OS 종류 | O | 라디오 버튼 | RedHat 계열, Ubuntu 계열, Windows, macOS 중 선택 |
| OS 버전 | O | 드롭다운 | OS 종류에 따라 목록 변경 |
| Python 버전 | O | 드롭다운 | 3.8 ~ 3.12 |

### 4.2 다운로드 입력

| 입력 항목 | 필수 | 입력 방식 | 검증 조건 |
|----------|-----|----------|----------|
| 환경 | O | 드롭다운 | 등록된 환경 중 선택 |
| 패키지 목록 | O | 파일 선택 또는 텍스트 입력 | 최소 1개 이상의 패키지 |
| 출력 폴더 | O | 폴더 선택 | 쓰기 권한이 있는 폴더 |
| 압축 여부 | X | 체크박스 | 기본값: 해제 |
| 분할 크기 | X | 숫자 입력 | 압축 선택 시에만 활성화. 기본값: 200MB |

---

## 5. 출력 명세

### 5.1 진행 중 출력

| 출력 항목 | 표시 방식 | 설명 |
|----------|----------|------|
| 전체 진행률 | 프로그레스 바 | 0~100% |
| 현재 작업 | 텍스트 | "다운로드 중: pandas (5/12)" |
| 패키지별 상태 | 스크롤 목록 | 각 패키지의 성공/실패/진행중 상태 |

### 5.2 완료 후 출력

| 출력 항목 | 설명 |
|----------|------|
| 총 패키지 수 | 요청한 패키지 + 의존성 패키지 |
| 성공 개수 | 다운로드 성공한 패키지 수 |
| 실패 개수 | 다운로드 실패한 패키지 수 |
| 총 용량 | 다운로드된 파일들의 총 크기 |
| 소요 시간 | 다운로드에 걸린 시간 |
| 출력 위치 | 결과물이 저장된 폴더 경로 |

### 5.3 상태 표시줄 메시지

| 상태 | 메시지 예시 |
|------|------------|
| 초기 | "준비됨" |
| 환경 선택됨 | "환경: prod-server (redhat-8, Python 3.12)" |
| 파일 선택됨 | "패키지 10개 로드됨" |
| 다운로드 중 | "다운로드 중... (5/12)" |
| 완료 | "완료: 12개 패키지 다운로드됨" |
| 오류 | "오류: requirements.txt를 찾을 수 없습니다" |

## 6. 인터랙션 상세

### 6.1 환경 선택 영역

| 동작 | 결과 |
|------|------|
| 드롭다운에서 환경 선택 | 하단에 환경 상세 정보 표시 |
| "추가" 버튼 클릭 | 환경 등록 팝업 열림 |
| "편집" 버튼 클릭 | 선택된 환경의 편집 팝업 열림 |
| "삭제" 버튼 클릭 | 확인 후 환경 삭제 |

### 6.2 패키지 입력 영역

| 동작 | 결과 |
|------|------|
| "찾아보기" 클릭 | 파일 선택 다이얼로그 열림 (.txt 필터) |
| 파일 드래그앤드롭 | 파일 경로 자동 입력, 내용 미리보기 |
| 텍스트 영역에 직접 입력 | 파일 선택 비활성화 |

### 6.3 다운로드 실행

| 동작 | 결과 |
|------|------|
| "다운로드" 클릭 | 입력 검증 후 다운로드 시작. 설정 영역 비활성화 |
| "취소" 클릭 | 확인 후 다운로드 중단. 이미 받은 파일은 유지 |
| 다운로드 완료 | 결과 팝업 표시. 설정 영역 다시 활성화 |

### 6.4 결과 팝업

| 동작 | 결과 |
|------|------|
| "폴더 열기" 클릭 | 시스템 파일 탐색기에서 출력 폴더 열림 |
| "리포트 보기" 클릭 | report.md 파일을 기본 뷰어로 열기 |
| "확인" 클릭 | 팝업 닫힘 |

## 7. 입력 검증

### 7.1 환경 등록 시

| 검증 항목 | 조건 | 오류 메시지 |
|----------|------|------------|
| 환경 이름 비어있음 | 필수 입력 | "환경 이름을 입력하세요" |
| 환경 이름 형식 | 영문, 숫자, -, _ | "환경 이름은 영문, 숫자, -, _ 만 사용 가능합니다" |
| 환경 이름 중복 | 기존 이름과 중복 | "이미 존재하는 환경 이름입니다" |

### 7.2 다운로드 시

| 검증 항목 | 조건 | 오류 메시지 |
|----------|------|------------|
| 환경 미선택 | 환경 선택 필수 | "환경을 선택하세요" |
| 패키지 없음 | 최소 1개 필요 | "패키지를 입력하거나 파일을 선택하세요" |
| 파일 없음 | 선택한 파일 존재 | "파일을 찾을 수 없습니다" |
| 출력 폴더 없음 | 폴더 선택 필수 | "출력 폴더를 선택하세요" |
| 쓰기 권한 없음 | 폴더 쓰기 가능 | "선택한 폴더에 쓰기 권한이 없습니다" |

## 8. 사용자 편의 기능

### 8.1 설정 기억
- 마지막으로 선택한 환경 기억
- 마지막으로 사용한 출력 폴더 경로 기억
- 압축 옵션 설정 기억

### 8.2 드래그앤드롭
- requirements.txt 파일을 메인 화면에 끌어다 놓으면 자동으로 파일 경로 입력

### 8.3 실시간 피드백
- OS/Python 선택 시 플랫폼 태그 즉시 미리보기
- 패키지 입력 시 파싱된 패키지 개수 표시
- 입력 오류 시 해당 필드에 빨간색 테두리와 메시지 표시

### 8.4 다운로드 중 보호
- 다운로드 진행 중에는 설정 변경 불가
- 프로그램 종료 시도 시 확인 팝업 표시

## 9. 성공 기준

| 항목 | 기준 |
|------|------|
| 사용 편의성 | CLI를 모르는 사용자도 5분 내에 첫 다운로드 성공 |
| 기능 동등성 | CLI의 모든 기능을 GUI에서 수행 가능 |
| 안정성 | 다운로드 중 취소해도 프로그램 정상 동작 |
| 피드백 | 모든 작업에 대해 시각적 피드백 제공 |
